Resumen del Módulo

El PromptTemplateManager permite que el personal administrativo modifique el tono, las reglas del negocio, el formato del reporte y las directrices de generación de imágenes en tiempo real, directamente desde la base de datos PostgreSQL, sin requerir despliegues de código.

Últimas Funciones Agregadas

  • Base de Datos PostgreSQL

    Esquema Prisma PromptTemplate persistido con clave única.

  • Controlador REST

    Endpoints GET /api/v1/prompts y PUT /api/v1/prompts/:key activos.

  • Resolución de Variables

    Mapeo y sustitución dinámica en caliente de variables tipo {{PLACEHOLDER}}.

Endpoints de Integración

Método Ruta Acción
GET /prompts Listar todos
GET /prompts/:key Obtener uno
PUT /prompts/:key Actualizar
Para validar, la documentación interactiva Swagger se encuentra en /api/docs.

Arquitectura del Flujo de Prompts

          sequenceDiagram
            actor Admin as Administrador (UI)
            participant Ctrl as PromptTemplateController
            participant Serv as PromptTemplateService
            participant DB as PostgreSQL (PromptTemplate)
            participant LLM as Gemini Module

            Admin->>Ctrl: PUT /prompts/:key (Nuevo Prompt)
            Ctrl->>Serv: update(key, data)
            Serv->>DB: Prisma Update
            DB-->>Serv: Confirmación
            Serv-->>Admin: Éxito (UI Actualizada)
            
            Note over Admin, LLM: En la siguiente ejecución del reporte...
            LLM->>Serv: resolvePrompt(key, variables)
            Serv->>DB: Carga plantilla de BD
            Serv->>LLM: Devuelve Prompt con placeholders sustituidos
            LLM->>LLM: Invocación a Gemini API
